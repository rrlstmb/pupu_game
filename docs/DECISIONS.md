# Decisions

ADR entries are append-only. Reversible assumptions are acceptable when recorded here.

| ID | Date | Decision | Reason | Cost | Reversible | Status |
|---|---|---|---|---|---|---|
| ADR-0001 | 2026-07-07 | Use Phaser 3 + TypeScript + Vite + Vitest + Playwright + npm. | Empty project; plan recommends Phaser or PixiJS; Phaser gives scene/input/audio/camera primitives and broad examples. | Phaser-specific adapter boundaries must be enforced. | Yes, before runtime code grows. | Accepted |
| ADR-0002 | 2026-07-07 | Phase 00 does not scaffold app code or package files. | User explicitly scoped Phase 00 to audit, decisions, and documents; gameplay and toolchain scaffold belong later. | No executable app yet. | Yes. | Accepted |
| ADR-0003 | 2026-07-07 | Use `src/domain` for pure rules and keep Phaser in scenes/adapters. | Required for deterministic tests and rule/display separation. | More explicit mapping code between domain and Phaser. | Difficult after gameplay grows, easy now. | Accepted |
| ADR-0004 | 2026-07-07 | Use injected seeded RNG for all rule-affecting randomness. | Throws, spawns, wind, and events must be reproducible and fair. | Every system needs RNG plumbing. | No, should remain invariant. | Accepted |
| ADR-0005 | 2026-07-07 | Use data-driven JSON-compatible definitions for levels, NPCs, poop, score, combo, and alert. | The plan requires tunable 10-level content and tactical differences. | Requires schema validation before authoring content at scale. | No, should remain invariant. | Accepted |
| ADR-0006 | 2026-07-07 | Roadmap uses 26 phases and 6 Release Gates. | User acceptance requires this structure; plan provides MVP, content, and release scale. | More phase docs to maintain. | Yes, by review before Phase 01. | Accepted |
| ADR-0007 | 2026-07-07 | Treat missing `AGENTS.md` and `docs` as Phase 00 work, not a blocker. | Phase 00 has no dependency and explicitly requires creating them. | First phase evidence must document absence. | Not needed after Phase 00. | Accepted |
| ADR-0008 | 2026-07-07 | Scaffold with npm, Vite dev server on `127.0.0.1`, strict TypeScript, ESLint, Vitest, and Playwright Chromium. | Provides a stable local browser game baseline and avoids exposing dev server outside localhost by default. | E2E requires Playwright browser download and local port permission in this sandbox. | Yes, before wider tooling grows. | Accepted |
| ADR-0009 | 2026-07-07 | Define world spatial language through a pure `WorldLayout` model and `Depths` constants. | Keeps scene rendering driven by one coordinate model and makes lanes/bounds testable across resolutions. | Scene drawing has an extra data mapping layer. | Yes, before gameplay entities depend on lane data. | Accepted |
| ADR-0010 | 2026-07-07 | Resolve simultaneous left/right input to neutral movement and clear input on blur/visibility/dispose. | Neutral resolution is deterministic and prevents hidden priority; clearing input prevents stuck keys after focus changes. | Players cannot intentionally override a held direction by pressing the other direction without releasing. | Yes, before advanced input customization. | Accepted |
| ADR-0011 | 2026-07-08 | Use canonical pixels, px/s velocity, and px/s^2 gravity/wind for projectile motion. | Matches Phaser rendering coordinates and keeps prediction/simulation directly comparable. | Future camera scaling must convert only at the presentation boundary. | No, should remain stable after golden cases. | Accepted |
| ADR-0012 | 2026-07-08 | Define landing as crossing a configured horizontal landing plane after entering the playable arc. | Supports throws launched from rooftop edge toward alley floor while avoiding immediate landing when origin is below a higher plane. | Sloped or object-specific collisions require later collision layers. | Yes, before collision phase. | Accepted |
| ADR-0013 | 2026-07-08 | NPC spawn overflow skips and counts failed spawns instead of queuing. | Prevents high-density events from creating unbounded backlog or burst allocation. | Spawn cadence can thin out under max-active pressure. | Yes, before authored high-density events. | Accepted |
| ADR-0014 | 2026-07-08 | Phase 06 collision uses projectile segment AABB overlap plus a `${projectileId}:${npcId}:${hitWindowId}` token. | Segment overlap prevents fast projectiles from tunneling through placeholder NPC bodies; hit tokens prevent duplicate collision callbacks from double-counting the same window. | Placeholder rectangular collision is coarse and may need per-poop/per-NPC collider data later. | Yes, before final collider tuning. | Accepted |
| ADR-0015 | 2026-07-08 | Scoring remains unimplemented and is reserved for `NPC_RANT_STARTED`, not `PROJECTILE_HIT`. | The product invariant says rant/reaction is the scoring moment; Phase 06 must not implement formal score early. | Future score systems must consume gameplay events rather than collision callbacks directly. | No, should remain an invariant. | Accepted |
| ADR-0016 | 2026-07-08 | Phase 07 score is calculated only from validated `NPC_RANT_STARTED` events and stored as immutable `ScoreBreakdown` entries. | Keeps UI from awarding points and makes each score source auditable. | Gameplay events need stable ids and scoring metadata. | No, should remain an invariant. | Accepted |
| ADR-0017 | 2026-07-08 | `Hit`, `Ranting`, and `Recovering` pause the combo timer as a hit-stop window. | Lets players read the reaction that caused scoring without losing combo time; prompt required hit-stop behavior to be explicit. | Combo duration feels more forgiving during repeated-hit loops. | Yes, before final pacing tuning. | Accepted |
| ADR-0018 | 2026-07-08 | Alert changes are source-recorded domain events on `AlertState`, not direct numeric mutation. | HUD/debug need to explain why alert changed, and tests need reproducible source accounting. | Per-frame changes require throttling so recent sources remain readable. | No, should remain an invariant. | Accepted |
| ADR-0019 | 2026-07-08 | Retry uses `scene.restart()` plus explicit GameScene state reinitialization in `create()`. | Phaser reuses scene instances; without explicit field reset, caught/score/tokens can leak after retry. | New runtime fields must be added to the reset block. | Yes, if a later level-session owner centralizes state reset. | Accepted |
| ADR-0020 | 2026-07-10 | Model tactical poop as data definitions plus capability strategies. | Phase 09 needs normal, sticky, splash, and jumbo poop to change decisions without embedding rule switches throughout collision code. | Strategy plumbing adds indirection for the small initial set. | Yes, before many more poop types ship. | Accepted |
| ADR-0021 | 2026-07-10 | Sticky refreshes a same-type slow effect, splash dedupes per effect instance, and jumbo only marks `breaksDefense` in Phase 09. | These rules make stacking and repeated hits testable while avoiding future-phase defense implementation. | Sticky cannot stack multiplicatively yet; defense destruction remains deferred. | Yes, before balance lock. | Accepted |
| ADR-0022 | 2026-07-10 | Store projectile config/rules on each active projectile. | Phase 10 has multiple projectile behaviors; active arcs must not change when the player switches arsenal selection. | Projectile state is larger and tests need richer fixtures. | No, should remain stable for deterministic throws. | Accepted |
| ADR-0023 | 2026-07-10 | Stink zones use a reusable `EnvironmentalEffectZone` domain state with lifecycle stats. | Future cleaner systems need a clearing interface, and Phase 10 needs zone lifetime/object leak evidence. | Full pathfinding is deferred; current fallback is slow-in-place rather than route recomputation. | Yes, before cleaner/route phases. | Accepted |
| ADR-0024 | 2026-07-10 | Phase 10 arsenal sandbox is debug-only Alt+number selection. | It lets reviewers exercise eight poop types without changing normal Q/E inventory controls or adding final UI. | Browser e2e must avoid timing assumptions around global projectile cooldown. | Yes, when final arsenal UI arrives. | Accepted |
| ADR-0025 | 2026-07-10 | Model advanced NPC behavior with tags and composed abilities plus an interaction matrix. | Phase 11 adds many NPC types; type switches in scene/collision code would be brittle. | Definitions and fixtures are more verbose. | No, should remain the extension model. | Accepted |
| ADR-0026 | 2026-07-10 | Missing NPC x poop matrix entries use safe default normal behavior and are reported by validation. | The full authored matrix will grow; development should not crash on missing pairs, but gaps must be visible. | Designers must review missing-pair reports before content lock. | Yes, before final matrix lock. | Accepted |
| ADR-0027 | 2026-07-14 | Treat `docs/CODEX_PHASE_PLAN.md` V2.0 as the source for the 26-prompt and Gate A-F dependency map. | Retro Gate A found that the standalone roadmap still described an obsolete phase split. | Roadmap, acceptance, and quality-gate summaries must be kept synchronized with the manual. | Yes, through a reviewed manual revision. | Accepted |
| ADR-0028 | 2026-07-14 | A scene disposer removes both shutdown/destroy hooks when either lifecycle event fires; input also clears on pause and resume. | Phaser reuses Scene instances, so an untriggered counterpart listener and held pause input can survive repeated lifecycle transitions. | Adapters register two additional scene listeners and lifecycle tests must account for Phaser's own baseline listeners. | No, this is a cleanup invariant. | Accepted |
| ADR-0029 | 2026-07-14 | Pool Phaser projectile views separately from pure projectile state. | Prompt 04 required object-pool reuse, while the adapter previously destroyed every recycled view. | The adapter tracks active/pooled/created/reused counts and must destroy both collections on shutdown. | Yes, if replaced by an equivalent bounded renderer pool. | Accepted |
| ADR-0030 | 2026-07-14 | Separate collision, rant start, and recovery into state-bound domain events. | Retro Gate B found collision emitted the scoring event before the NPC actually entered `Ranting`, while `NPC_RECOVERED` was never emitted. | NPC state carries pending immutable rant context and the scene compares consecutive state snapshots. | No, event causality is a core-loop invariant. | Accepted |
| ADR-0031 | 2026-07-14 | Make scoring idempotent by gameplay `eventId` and retire hit tokens with recycled projectiles. | Replayed events and indefinitely retained tokens could respectively duplicate score or grow session memory. | Score lookup and token pruning add small linear work over bounded session collections. | Yes, if replaced by equivalent indexed deduplication. | Accepted |
| ADR-0032 | 2026-07-14 | Pool Phaser NPC views and treat retry as complete Core Loop session reinitialization. | Retro Gate B found NPC views were destroyed on exit and retry preserved alert history plus debug wind/config. | New runtime fields must join the reset contract; pool statistics are exposed for acceptance. | Yes, if a later LevelSession owner provides equivalent cleanup. | Accepted |
| ADR-0033 | 2026-07-14 | Inject alert stage thresholds through `AlertRules`. | Hard-coded boundaries contradicted the data-driven alert requirement and prevented alternate rule fixtures. | Every stage calculation must receive rules explicitly. | No, alert tuning must remain data driven. | Accepted |

## Unknowns and Reversible Defaults

- Phaser, Vite, Vitest, Playwright, TypeScript, and ESLint versions are now locked by `package-lock.json`.
- npm script names are finalized for Phase 01 in `package.json`.
- CI provider is unknown; Phase 01 should keep scripts provider-agnostic.
- Asset pipeline is unknown; use placeholder assets only when validating presentation, never rule completion.
- Browser support targets are unknown; default to current Chromium for Playwright smoke until product targets are specified.
